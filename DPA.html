1. Subject of the Agreement

This Data Processing Agreement (“DPA”) governs the processing of personal data by [CompanyName] GmbH (in Gründung] on behalf of the Controller when using the service [ProductName – e.g., FileBoss].

The purpose is to ensure full compliance with Art. 28 GDPR.

2. Nature and Purpose of Processing

We process personal data solely for the following purposes:

  • Operating the FileBoss platform
  • File classification, renaming, and organizing
  • Google Drive integration
  • Communication between user and platform
  • Technical operation, security, and maintenance
  • Subscription management and payment processing

No other processing will be carried out.

3. Type of Personal Data

3.1 User Account Data

  • Name
  • Email
  • Profile picture
  • Language, timezone

3.2 Files & Google Drive Metadata

  • File names
  • Drive folder IDs
  • Document metadata
  • Content required for classification

3.3 Technical Data

  • IP address
  • Device data
  • Logs
  • AI usage information

3.4 Payment Data

  • Billing email
  • Subscription data
  • Provider payment IDs

4. Categories of Data Subjects

  • Users of the platform
  • Employees of the Controller
  • Suppliers / Clients appearing in uploaded documents
  • Any persons mentioned in processed files

5. Duration of Processing

The processing continues until the Controller deletes their account or until termination of the DPA or Terms of Service.

6. Obligations of the Processor

6.1 Process Only Under Instructions

We process data only on documented instructions from the Controller.

6.2 Confidentiality

All employees and subcontractors are legally bound by confidentiality.

6.3 Security Measures

We implement strong technical and organizational measures, including:

  • Encryption (TLS/HTTPS and at-rest encryption)
  • Access control
  • Secure authentication
  • Two-factor support
  • Security audits
  • Restricted internal access
  • Token protection

6.4 Cooperation with Data Subjects

We assist with:

  • Access requests
  • Corrections
  • Deletion
  • Objections
  • Data portability

6.5 Breach Notification

In case of a breach, the Processor will notify the Controller without undue delay (Art. 33 GDPR).

7. Obligations of the Controller

  • Ensure lawful basis for uploaded data
  • Maintain secure Google Drive access
  • Provide correct instructions
  • Follow data deletion policies
  • Notify us of invalid or revoked OAuth access

8. Sub-Processors

We may use third-party service providers ("Sub-processors"), including:

  • Supabase — database, auth, storage
  • Google Cloud / Drive API — storage access
  • Vercel — hosting
  • Stripe / Apple / Google — payments
  • Email providers

All Sub-processors comply with Art. 28 GDPR.

9. International Data Transfers

  • Standard Contractual Clauses (SCCs)
  • Adequacy decisions
  • Additional technical safeguards

10. Deletion and Return of Data

Upon termination:

  • All personal data will be deleted
  • Or returned if requested
  • Backups purged automatically
  • Google tokens revoked

11. Audits

The Controller may:

  • Request information
  • Request summaries of audits
  • Review our security documentation

12. Liability

Both parties are liable only to the extent permitted under GDPR and the Terms of Service.

13. Termination

This DPA remains in effect while the Controller uses the service and until all personal data is deleted or returned.

14. Final Provisions

If any part of this DPA becomes invalid, the remaining provisions remain in effect.
